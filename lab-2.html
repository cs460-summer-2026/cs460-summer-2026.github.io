Read this first

Work through the steps in order. Each step tells you what to type and what you should see afterward. If something looks different, have a look at Troubleshooting at the bottom, your issue is most likely there with a quick fix.

Three things that save you grief: write down the root password you set, always open a new terminal after changing settings, and don't skip the "what you should see" checks.

1. Install MySQL

This installs the database server itself. Download MySQL from dev.mysql.com/downloads/mysql. When Oracle asks you to sign in, scroll down and click "No thanks, just start my download".

Somewhere during install you will be asked to set a root password. Type one you will remember and write it down. You need it in Step 3.

macOS

Pick the ARM64 DMG if your Mac is Apple Silicon (M1/M2/M3/M4), or x86 if it's an older Intel Mac. Not sure? Apple menu → About This Mac → look at "Chip". Open the DMG, run the installer, keep the defaults.

If you have Homebrew, this is faster:

brew install mysql
brew services start mysql
mysql_secure_installation

The last command sets your root password. (With Homebrew you can also just leave the password blank and press Enter at every prompt for this lab.)

Linux (Ubuntu/Debian)

sudo apt update
sudo apt install mysql-server
sudo mysql_secure_installation

Windows

Download the MySQL Installer for Windows and choose Developer Default. That installs the server, Workbench, and the Python connector in one go, so you can skip Step 2.

Check it worked

Open a terminal (Mac: Terminal app; Windows: PowerShell) and run:

mysql --version

You should see a version number like mysql Ver 9.x.

Didn't work? Common Step 1 problems: wrong architecture, macOS blocked the installer, Homebrew formula not found, Windows service won't start, command not found.

2. Install MySQL Workbench

Workbench is the app you'll actually use: you type SQL in an editor, click run, and see results in a grid. Download it from dev.mysql.com/downloads/workbench.

macOS

Open the DMG and drag MySQL Workbench into the Applications folder. The first time you open it, macOS may say it's from an "unidentified developer". If so: System Settings → Privacy & Security, scroll to the Security section, click Open Anyway, then confirm.

Linux

Install the .deb from the download page, or use your package manager.

Windows

Already installed if you chose Developer Default in Step 1. Otherwise download the MSI from the link above.

Check it worked

MySQL Workbench opens and shows a home screen with a "MySQL Connections" area.

Didn't work? Workbench crashes on launch (Apple Silicon), macOS says it's "damaged".

3. Connect to your server

On the Workbench home screen you'll see a connection tile, usually labeled Local instance 3306. Double-click it.

  1. Workbench asks for the root password. Type the one you set in Step 1. If you installed with Homebrew and left it blank, leave the field empty.
  2. Tick Save password in keychain so it stops asking.
  3. Click OK.

You may see a yellow warning that your MySQL version is newer than Workbench officially supports. Click OK and ignore it. Everything in this lab works fine. See Troubleshooting: version warning for the why.

Check it worked

A new tab opens with three areas: a Navigator on the left (showing SCHEMAS), an empty SQL editor in the middle, and a results area at the bottom.

Didn't work? Access denied (wrong password), can't connect (server not running), could not acquire management access, no connection tile, version warning.

4. Create the database

You'll paste each block below into the Workbench SQL editor, then click the lightning-bolt button (⚡) to run it. The bolt that runs everything in the editor is the first one; you can also press Cmd+Shift+Enter (Mac) or Ctrl+Shift+Enter (Windows).

4a. Make the database

CREATE DATABASE space_missions;
USE space_missions;

In the Navigator on the left, click the refresh icon next to SCHEMAS. space_missions appears. Double-click it so its name goes bold, which means it's the active database.

4b. Create the tables

CREATE TABLE spaceships (
    sid            INT NOT NULL PRIMARY KEY AUTO_INCREMENT,
    spaceship_name VARCHAR(255),
    capacity       INT,
    type           VARCHAR(255)
);

CREATE TABLE destinations (
    did              INT NOT NULL PRIMARY KEY,
    destination_name VARCHAR(255),
    size             ENUM('S', 'M', 'L'),
    type             VARCHAR(255)
);

CREATE TABLE missions (
    sid         INT NOT NULL PRIMARY KEY,
    did         INT NOT NULL,
    description TEXT,
    duration    TEXT
);
Check it worked

The Action Output panel at the bottom shows three green checkmarks. Expand space_missions → Tables in the Navigator and you'll see destinations, missions, spaceships.

4c. Add the foreign keys

The missions table references the other two, so we mark its columns as foreign keys:

ALTER TABLE missions ADD FOREIGN KEY (sid) REFERENCES spaceships(sid);
ALTER TABLE missions ADD FOREIGN KEY (did) REFERENCES destinations(did);

4d. Insert some data

INSERT INTO spaceships (spaceship_name, capacity, type)
VALUES ('Explorer', 5, 'Scientific');

INSERT INTO destinations (did, destination_name, size, type)
VALUES (100, 'Jupiter', 'L', 'Planet');

INSERT INTO missions (sid, did, description, duration)
VALUES (1, 100, 'Research', '3 Years');

This records that one spaceship, Explorer, was sent to Jupiter for a 3-year research mission.

4e. Test query

SELECT s.spaceship_name, d.destination_name, m.description
FROM   spaceships s, destinations d, missions m
WHERE  s.sid = m.sid
  AND  d.did = m.did;
Check it worked

The bottom of the editor shows a results grid with one row:

Explorer | Jupiter | Research

If you see that, your database is set up correctly.

Empty result usually means the INSERT step (4d) didn't run, re-run it, then the query. "Table already exists" means you ran 4b twice, which is harmless.

5. Python connector

Every lab after this one uses Python to query MySQL. The connector is the library that lets your Python code talk to the database. We test it now so a broken setup gets fixed today, not the night before homework is due.

Check Python 3 is installed first:

python3 --version

If that prints a version (3.9 or newer), continue. If not, install Python from python.org/downloads.

5a. Make a virtual environment

A virtual environment is a private, throwaway Python folder where you can install libraries without touching the rest of your system. Modern Python (and Homebrew) require this on many machines, so we use it from the start.

In a terminal:

macOS / Linux

python3 -m venv cs460env
source cs460env/bin/activate

Windows (PowerShell)

python -m venv cs460env
cs460env\Scripts\activate
Check it worked

Your terminal prompt now starts with (cs460env). That means you're inside the environment. You'll need to run the activate line again any time you open a new terminal.

5b. Install the connector

pip install mysql-connector-python

Because you're inside the virtual environment, this just works. If you skipped Step 5a and see an error about an externally-managed-environment, that's exactly why the environment exists, see Troubleshooting: externally-managed-environment.

5c. Write the test script

Do not use TextEdit's default mode, it saves rich text and Python can't read it. The safest way is to create the file from the terminal. Paste this whole block:

cat > test_mysql.py << 'EOF'
import mysql.connector

db = mysql.connector.connect(
    user='root',
    host='127.0.0.1',
    port=3306,
    password='',
    database='space_missions'
)

cursor = db.cursor()

query = """
SELECT s.spaceship_name, d.destination_name, m.description
FROM   spaceships s, destinations d, missions m
WHERE  s.sid = m.sid
  AND  d.did = m.did
"""

cursor.execute(query)
for row in cursor:
    print(row)

cursor.close()
db.close()
EOF

If you set a root password in Step 1, open test_mysql.py and put it between the quotes on the password='' line. If your password is blank, leave it as is.

(Prefer an editor? Use VS Code, or in TextEdit choose Format → Make Plain Text first, and save the file named exactly test_mysql.py.)

5d. Run it

python test_mysql.py
Check it worked 
('Explorer', 'Jupiter', 'Research')

That one tuple means Python connected to MySQL, ran the query, and got the result back. The connector works and the lab is complete.

Didn't work? externally-managed-environment, PowerShell blocks venv activation, No module named 'mysql', cryptography is required, caching_sha2_password / RSA key, file not found (TextEdit .rtf), access denied from Python only.

You're done when…

  1. mysql --version prints a version number.
  2. MySQL Workbench connects to your local server.
  3. The test query in Step 4e returns Explorer | Jupiter | Research.
  4. Running python test_mysql.py prints ('Explorer', 'Jupiter', 'Research').

If all four are true, you've finished Lab 2. Keep your root password somewhere safe, you'll use this same setup for the rest of the course.

Troubleshooting

Find your error below. It's grouped by which step you're on. Your issue is almost certainly here, and the fix is short. If you've worked through it and you're still stuck, feel free to ask.

Installing MySQL (Step 1)

"MySQL can't be installed on this computer" / installer won't run

You downloaded the wrong architecture. On a Mac, ARM64 is for Apple Silicon (M1 through M4) and x86 is for older Intel Macs. Check yours under Apple menu → About This Mac → "Chip", then download the matching DMG.

macOS: "cannot be opened because Apple cannot check it for malicious software"

macOS Gatekeeper is blocking an unsigned installer. Go to System Settings → Privacy & Security, scroll to the Security section, and click Open Anyway, then confirm. Re-run the installer.

Homebrew: "No available formula with the name 'mysql'"

Your Homebrew is out of date. Update it and try again:

brew update
brew install mysql

Windows: "Could not start the service MySQL" during install

Usually a leftover from a previous MySQL install, or the service name is taken. In the MySQL Installer, remove any old MySQL products, reboot, and reinstall. If it persists, give the service a fresh name (for example MySQL84) on the configuration screen.

"command not found: mysql" (terminal can't find it)

MySQL is installed but not on your PATH. This only matters if you use the terminal; Workbench works either way.

macOS (DMG install)

echo 'export PATH="/usr/local/mysql/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

macOS (Homebrew)

Should already be on PATH. If not, run this and open a new terminal:

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile

Windows

Search "Environment Variables" → Edit the system environment variables → Environment Variables → under System variables select Path → Edit → New → add C:\Program Files\MySQL\MySQL Server 8.4\bin → OK out. Open a fresh terminal afterward.

Opening MySQL Workbench (Step 2)

Workbench crashes on launch or when opening a diagram (Apple Silicon)

Older Workbench builds crash on M-series Macs. Install the latest version from the download page, that fixes it for most people. If a fresh install still crashes, the terminal client (Step 6 alternative) works as a fallback.

macOS: Workbench "is damaged and can't be opened"

Gatekeeper again. Same fix as the installer: System Settings → Privacy & Security → Open Anyway. If macOS won't show the button, re-download from the official link above rather than a mirror.

Connecting to the server (Step 3)

Yellow warning: "Incompatible / nonstandard server version (9.x)"

Workbench was last built for MySQL 8.0, your server is newer. Everything in this course works fine. Click OK and continue. Only advanced migration features are affected, which we never use.

"ERROR 1045: Access denied for user 'root'"

Wrong password. Retype the one you set in Step 1, watching for caps lock. If you installed with Homebrew and never set one, leave the field blank. If you genuinely forgot it, the simplest fix in a lab setting is to uninstall and reinstall MySQL with a password you'll remember.

"Can't connect to MySQL server on '127.0.0.1'"

The server isn't running. Start it:

macOS (Homebrew)

brew services start mysql

macOS (DMG)

System Settings → MySQL → Start MySQL Server.

Linux

sudo systemctl start mysql

Windows

net start MySQL

Workbench: "Could not acquire management access for administration"

Workbench connected to the database but can't manage the server process, usually because the connection's settings don't match how the server actually runs. For this lab you don't need management access, you can dismiss it and still run SQL. If you want it gone, edit the connection (the wrench/"Manage Connections") and make sure Hostname is 127.0.0.1 and Port is 3306.

No "Local instance" tile on the Workbench home screen

Create the connection by hand. Click the "+" next to MySQL Connections and fill in: Hostname 127.0.0.1, Port 3306, Username root. Click "Test Connection", then OK.

"ERROR 1130: Host is not allowed to connect to this MySQL server"

You're connecting from a host the server doesn't permit. Use 127.0.0.1 as the host (not your machine name or an external IP). For a local lab this is all you need.

Python connector (Step 5)

"error: externally-managed-environment"

Your Python blocks system-wide installs to protect itself. This is exactly why we use a virtual environment. Make sure your prompt shows (cs460env) before running pip. If it doesn't, activate it again:

source cs460env/bin/activate    # macOS / Linux
cs460env\Scripts\activate       # Windows

Windows: "running scripts is disabled on this system" when activating the venv

PowerShell blocks scripts by default. Allow them for your account, then activate again:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Or sidestep it entirely by using Command Prompt (cmd) instead of PowerShell, cmd doesn't block scripts.

"ModuleNotFoundError: No module named 'mysql'"

You installed the connector with one Python but ran the script with a different one. Install and run from inside the same activated environment:

source cs460env/bin/activate
pip install mysql-connector-python
python test_mysql.py

"cryptography is required for sha256_password or caching_sha2_password"

MySQL 8+ encrypts your password with a library Python doesn't ship by default. Install it inside your environment:

pip install cryptography

"RSA public key is not available" / "Authentication plugin caching_sha2_password could not be loaded"

An older connector talking to MySQL 8+ auth. Upgrade the connector inside your environment:

pip install --upgrade mysql-connector-python

If it still refuses, add auth_plugin='mysql_native_password' to the connect(...) call in your script.

"can't open file … No such file or directory" (but the file is right there)

TextEdit saved your script as test_mysql.py.rtf instead of test_mysql.py. Delete it and recreate it with the cat command in Step 5c, or in TextEdit choose Format → Make Plain Text before saving.

Python: access denied, but Workbench connects fine

The script's password='' line doesn't match your actual root password. Open test_mysql.py and put your password between the quotes. If your password is blank, leave it empty.

General

"Address already in use" (port 3306)

Another MySQL server is already running. You don't need a second one. Stop the extra instance, or just connect to the one that's already up.

Install fails completely on a managed / locked-down laptop

Some work or school laptops block installers. Two fallbacks: run MySQL in Docker (docker run --name cs460-mysql -e MYSQL_ROOT_PASSWORD=cs460pw -p 3306:3306 -d mysql:8.4), or do today's exercises on a classmate's machine and finish the install at home before Lab 3.
