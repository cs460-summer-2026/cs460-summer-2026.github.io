1. Install MySQL

Download MySQL 8.4 LTS from dev.mysql.com/downloads/mysql. You can skip the Oracle account when prompted.

The installer will ask you to set a root password. Pick something you will remember. You need it every time you connect.

macOS

Open the DMG and follow the prompts. Pick ARM 64 for Apple Silicon, x86 64 for Intel.

Or with Homebrew:

brew install mysql
brew services start mysql
mysql_secure_installation

Linux (Ubuntu/Debian)

sudo apt update
sudo apt install mysql-server
sudo mysql_secure_installation

Windows

Download the MySQL Installer for Windows. Choose Developer Default. That installs the server, the shell, and the Python connector together.

2. MySQL Workbench

Workbench is the official GUI. You write SQL in an editor, click run, see results in a grid. We use it for every lab from here on.

Download from dev.mysql.com/downloads/workbench.

macOS

Open the DMG and drag MySQL Workbench into Applications. If macOS refuses to open it because it's from an "unidentified developer", go to System Settings → Privacy & Security, scroll down, and click Open Anyway.

Linux

Install from the .deb on the download page, or via your package manager.

Windows

If you chose Developer Default in Section 1, Workbench is already installed. Otherwise download the MSI from the link above.

3. Python connector

Check Python is installed: python3 --version. Then:

pip install mysql-connector-python

Anaconda users:

conda install -c anaconda mysql-connector-python

Confirm it imports:

python3 -c "import mysql.connector; print(mysql.connector.__version__)"

4. Set PATH

Run mysql -V in a fresh terminal. If you see a version number, skip this section.

macOS (DMG install)

echo 'export PATH="/usr/local/mysql/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

macOS (Homebrew)

Homebrew adds /opt/homebrew/bin to PATH for you on Apple Silicon. If it didn't:

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile

Open a new terminal.

Windows

  1. Start → search "Environment Variables" → open Edit the system environment variables.
  2. Click Environment Variables.
  3. Under System variables, select Path, click Edit.
  4. Click New, paste C:\Program Files\MySQL\MySQL Server 8.4\bin.
  5. OK out of everything. Open a fresh PowerShell.

5. Start the server

The installer usually starts the server for you. If Section 6 works, ignore this. Otherwise:

macOS

Homebrew install:

brew services start mysql
brew services stop mysql

DMG install: System Settings → MySQL, or:

sudo /usr/local/mysql/support-files/mysql.server start

Linux

sudo systemctl start mysql
sudo systemctl stop mysql

Windows

net start MySQL
net stop MySQL

Default service name is MySQL. If you set a custom name during install, look it up in services.msc.

6. Connect

Open MySQL Workbench. On the home screen you'll see a tile labeled Local instance 3306 (or similar). Click it.

Workbench prompts for the root password. Type the one you set in Section 1 (or leave it blank if you didn't set one) and click OK. You should land in the SQL editor with a sidebar showing schemas on the left.

If you'd rather use the terminal, run:

mysql -u root -h 127.0.0.1 -p

Nothing shows on screen while you type the password. Press Enter. You should see a welcome banner and the mysql> prompt.

7. Create a database

Open MySQL Workbench. Click the default Local instance connection tile. Enter your password (blank if you didn't set one). You should see a SQL editor tab.

Paste each block below into the editor and click the lightning-bolt button (or press Cmd+Enter / Ctrl+Enter) to run.

Create the database

CREATE DATABASE space_missions;
SHOW DATABASES;
USE space_missions;

SHOW DATABASES confirms it exists. USE switches into it for the rest of the session.

Create tables

CREATE TABLE spaceships (
    sid            INT NOT NULL PRIMARY KEY AUTO_INCREMENT,
    spaceship_name VARCHAR(255),
    capacity       INT,
    type           VARCHAR(255)
);

CREATE TABLE destinations (
    did              INT NOT NULL PRIMARY KEY,
    destination_name VARCHAR(255),
    size             ENUM('S', 'M', 'L'),
    type             VARCHAR(255)
);

CREATE TABLE missions (
    sid         INT NOT NULL PRIMARY KEY,
    did         INT NOT NULL,
    description TEXT,
    duration    TEXT
);

Add foreign keys

ALTER TABLE missions ADD FOREIGN KEY (sid) REFERENCES spaceships(sid);
ALTER TABLE missions ADD FOREIGN KEY (did) REFERENCES destinations(did);

Insert data

INSERT INTO spaceships (spaceship_name, capacity, type)
VALUES ('Explorer', 5, 'Scientific');

INSERT INTO destinations (did, destination_name, size, type)
VALUES (100, 'Jupiter', 'L', 'Planet');

INSERT INTO missions (sid, did, description, duration)
VALUES (1, 100, 'Research', '3 Years');

One spaceship, Explorer, sent to Jupiter for a 3-year research mission.

Test it

SELECT s.spaceship_name, d.destination_name, m.description
FROM   spaceships s, destinations d, missions m
WHERE  s.sid = m.sid
  AND  d.did = m.did;

If the result is Explorer | Jupiter | Research, your database works.

Practice SQL elsewhere: SQLBolt, SQLZoo, or the W3Schools editor.

8. Python (optional)

Save this as test_mysql.py and replace #PASSWORD# with your root password.

import mysql.connector

db = mysql.connector.connect(
    user='root',
    host='127.0.0.1',
    port=3306,
    password='#PASSWORD#',
    database='space_missions'
)

cursor = db.cursor()

query = """
SELECT s.spaceship_name, d.destination_name, m.description
FROM   spaceships s, destinations d, missions m
WHERE  s.sid = m.sid
  AND  d.did = m.did
"""

cursor.execute(query)
for row in cursor:
    print(row)

cursor.close()
db.close()

Run it:

python3 test_mysql.py

You should see ('Explorer', 'Jupiter', 'Research'). If you do, the connector works and you're done.

Troubleshooting

ErrorCauseFix
ERROR 1045: Access denied for user 'root' Wrong password. Retype it. If you forgot it, reinstall MySQL.
Can't connect to MySQL server on '127.0.0.1' Server isn't running. See Section 5.
command not found: mysql PATH not set. Section 4. Use a fresh terminal after.
Address already in use (port 3306) Another MySQL is already running. Stop the old one first.
ModuleNotFoundError: No module named 'mysql' Connector installed for a different Python. python3 -m pip install mysql-connector-python
authentication plugin 'caching_sha2_password' cannot be loaded Old client, new MySQL. pip install --upgrade mysql-connector-python
