Read this first

Work through the steps in order. Each step tells you what to type and what you should see afterward. If something looks different, check Troubleshooting at the bottom before raising your hand. Most problems are listed there with a one-line fix.

Three things that save you grief: write down the root password you set, always open a new terminal after changing settings, and don't skip the "what you should see" checks.

1. Install MySQL

This installs the database server itself. Download MySQL from dev.mysql.com/downloads/mysql. When Oracle asks you to sign in, scroll down and click "No thanks, just start my download".

Somewhere during install you will be asked to set a root password. Type one you will remember and write it down. You need it in Step 3.

macOS

Pick the ARM64 DMG if your Mac is Apple Silicon (M1/M2/M3/M4), or x86 if it's an older Intel Mac. Not sure? Apple menu → About This Mac → look at "Chip". Open the DMG, run the installer, keep the defaults.

If you have Homebrew, this is faster:

brew install mysql
brew services start mysql
mysql_secure_installation

The last command sets your root password. (With Homebrew you can also just leave the password blank and press Enter at every prompt for this lab.)

Linux (Ubuntu/Debian)

sudo apt update
sudo apt install mysql-server
sudo mysql_secure_installation

Windows

Download the MySQL Installer for Windows and choose Developer Default. That installs the server, Workbench, and the Python connector in one go, so you can skip Step 2.

Check it worked

Open a terminal (Mac: Terminal app; Windows: PowerShell) and run:

mysql --version

You should see a version number like mysql Ver 9.x. If you get "command not found", see Troubleshooting: command not found.

2. Install MySQL Workbench

Workbench is the app you'll actually use: you type SQL in an editor, click run, and see results in a grid. Download it from dev.mysql.com/downloads/workbench.

macOS

Open the DMG and drag MySQL Workbench into the Applications folder. The first time you open it, macOS may say it's from an "unidentified developer". If so: System Settings → Privacy & Security, scroll to the Security section, click Open Anyway, then confirm.

Linux

Install the .deb from the download page, or use your package manager.

Windows

Already installed if you chose Developer Default in Step 1. Otherwise download the MSI from the link above.

Check it worked

MySQL Workbench opens and shows a home screen with a "MySQL Connections" area.

3. Connect to your server

On the Workbench home screen you'll see a connection tile, usually labeled Local instance 3306. Double-click it.

  1. Workbench asks for the root password. Type the one you set in Step 1. If you installed with Homebrew and left it blank, leave the field empty.
  2. Tick Save password in keychain so it stops asking.
  3. Click OK.

You may see a yellow warning that your MySQL version is newer than Workbench officially supports. Click OK and ignore it. Everything in this lab works fine. See Troubleshooting: version warning for the why.

Check it worked

A new tab opens with three areas: a Navigator on the left (showing SCHEMAS), an empty SQL editor in the middle, and a results area at the bottom.

Two common stumbles here: a wrong password gives Access denied (fix), and a server that isn't running gives Can't connect (fix).

4. Create the database

You'll paste each block below into the Workbench SQL editor, then click the lightning-bolt button (⚡) to run it. The bolt that runs everything in the editor is the first one; you can also press Cmd+Shift+Enter (Mac) or Ctrl+Shift+Enter (Windows).

4a. Make the database

CREATE DATABASE space_missions;
USE space_missions;

In the Navigator on the left, click the refresh icon next to SCHEMAS. space_missions appears. Double-click it so its name goes bold, which means it's the active database.

4b. Create the tables

CREATE TABLE spaceships (
    sid            INT NOT NULL PRIMARY KEY AUTO_INCREMENT,
    spaceship_name VARCHAR(255),
    capacity       INT,
    type           VARCHAR(255)
);

CREATE TABLE destinations (
    did              INT NOT NULL PRIMARY KEY,
    destination_name VARCHAR(255),
    size             ENUM('S', 'M', 'L'),
    type             VARCHAR(255)
);

CREATE TABLE missions (
    sid         INT NOT NULL PRIMARY KEY,
    did         INT NOT NULL,
    description TEXT,
    duration    TEXT
);
Check it worked

The Action Output panel at the bottom shows three green checkmarks. Expand space_missions → Tables in the Navigator and you'll see destinations, missions, spaceships.

4c. Add the foreign keys

The missions table references the other two, so we mark its columns as foreign keys:

ALTER TABLE missions ADD FOREIGN KEY (sid) REFERENCES spaceships(sid);
ALTER TABLE missions ADD FOREIGN KEY (did) REFERENCES destinations(did);

4d. Insert some data

INSERT INTO spaceships (spaceship_name, capacity, type)
VALUES ('Explorer', 5, 'Scientific');

INSERT INTO destinations (did, destination_name, size, type)
VALUES (100, 'Jupiter', 'L', 'Planet');

INSERT INTO missions (sid, did, description, duration)
VALUES (1, 100, 'Research', '3 Years');

This records that one spaceship, Explorer, was sent to Jupiter for a 3-year research mission.

4e. Test query

SELECT s.spaceship_name, d.destination_name, m.description
FROM   spaceships s, destinations d, missions m
WHERE  s.sid = m.sid
  AND  d.did = m.did;
Check it worked

The bottom of the editor shows a results grid with one row:

Explorer | Jupiter | Research

If you see that, your database is set up correctly.

5. Python connector

Every lab after this one uses Python to query MySQL. The connector is the library that lets your Python code talk to the database. We test it now so a broken setup gets fixed today, not the night before homework is due.

Check Python 3 is installed first:

python3 --version

If that prints a version (3.9 or newer), continue. If not, install Python from python.org/downloads.

5a. Make a virtual environment

A virtual environment is a private, throwaway Python folder where you can install libraries without touching the rest of your system. Modern Python (and Homebrew) require this on many machines, so we use it from the start.

In a terminal:

macOS / Linux

python3 -m venv cs460env
source cs460env/bin/activate

Windows (PowerShell)

python -m venv cs460env
cs460env\Scripts\activate
Check it worked

Your terminal prompt now starts with (cs460env). That means you're inside the environment. You'll need to run the activate line again any time you open a new terminal.

5b. Install the connector

pip install mysql-connector-python

Because you're inside the virtual environment, this just works. If you skipped Step 5a and see an error about an externally-managed-environment, that's exactly why the environment exists, see Troubleshooting: externally-managed-environment.

5c. Write the test script

Do not use TextEdit's default mode, it saves rich text and Python can't read it. The safest way is to create the file from the terminal. Paste this whole block:

cat > test_mysql.py << 'EOF'
import mysql.connector

db = mysql.connector.connect(
    user='root',
    host='127.0.0.1',
    port=3306,
    password='',
    database='space_missions'
)

cursor = db.cursor()

query = """
SELECT s.spaceship_name, d.destination_name, m.description
FROM   spaceships s, destinations d, missions m
WHERE  s.sid = m.sid
  AND  d.did = m.did
"""

cursor.execute(query)
for row in cursor:
    print(row)

cursor.close()
db.close()
EOF

If you set a root password in Step 1, open test_mysql.py and put it between the quotes on the password='' line. If your password is blank, leave it as is.

(Prefer an editor? Use VS Code, or in TextEdit choose Format → Make Plain Text first, and save the file named exactly test_mysql.py.)

5d. Run it

python test_mysql.py
Check it worked 
('Explorer', 'Jupiter', 'Research')

That one tuple means Python connected to MySQL, ran the query, and got the result back. The connector works and the lab is complete.

If you get No module named 'mysql', your pip and python don't match, see Troubleshooting: No module named mysql.

You're done when…

  1. mysql --version prints a version number.
  2. MySQL Workbench connects to your local server.
  3. The test query in Step 4e returns Explorer | Jupiter | Research.
  4. Running python test_mysql.py prints ('Explorer', 'Jupiter', 'Research').

If all four are true, you've finished Lab 2. Keep your root password somewhere safe, you'll use this same setup for the rest of the course.

Troubleshooting

Find your error message below. If yours isn't here, raise your hand.

"command not found: mysql"

MySQL is installed but your terminal doesn't know where it is. You need to add it to your PATH.

macOS (DMG install)

echo 'export PATH="/usr/local/mysql/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

macOS (Homebrew)

Should already be on PATH. If not, run eval "$(/opt/homebrew/bin/brew shellenv)" and open a new terminal.

Windows

Search "Environment Variables" → Edit the system environment variables → Environment Variables → under System variables select Path → Edit → New → add C:\Program Files\MySQL\MySQL Server 8.4\bin → OK out. Open a fresh terminal afterward.

Yellow warning: "Incompatible/nonstandard server version (9.x)"

Workbench was last updated for MySQL 8.0, but your server is 9.x. Everything you do in this course works fine. Click OK and continue. Only advanced migration features are affected, which we never use.

"ERROR 1045: Access denied for user 'root'"

Wrong password. Retype the one you set in Step 1. If you genuinely forgot it, the simplest fix is to uninstall and reinstall MySQL, setting a password you'll remember this time.

"Can't connect to MySQL server on '127.0.0.1'"

The server isn't running. Start it:

macOS (Homebrew)

brew services start mysql

macOS (DMG)

System Settings → MySQL → Start MySQL Server.

Linux

sudo systemctl start mysql

Windows

net start MySQL

"error: externally-managed-environment"

Your Python refuses to install libraries system-wide to protect itself. This is exactly what the virtual environment in Step 5a is for. Make sure your prompt shows (cs460env) before running pip. If it doesn't, run the activate command again:

source cs460env/bin/activate    # macOS / Linux
cs460env\Scripts\activate       # Windows

"ModuleNotFoundError: No module named 'mysql'"

You installed the connector with one Python but ran the script with a different one. The fix is to install and run from inside the same virtual environment. Activate it, reinstall, then run:

source cs460env/bin/activate
pip install mysql-connector-python
python test_mysql.py

"can't open file … No such file or directory" (but the file exists)

TextEdit saved your script as test_mysql.py.rtf instead of test_mysql.py. Delete it and recreate the file using the cat command in Step 5c, or in TextEdit choose Format → Make Plain Text before saving.

"Address already in use" (port 3306)

Another MySQL server is already running. You don't need a second one. Stop the extra instance, or just connect to the one that's already up.
