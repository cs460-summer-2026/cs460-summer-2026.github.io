Read this first

Goal: get the VERSUS skeleton running at localhost:5000 where you can register, log in, create a bracket, and view it. That's the starting point for PA1.

You need MySQL working from Lab 2. If it isn't, fix that with Lab 2 Troubleshooting before continuing.

If you hit an error, find it in Troubleshooting at the bottom.

Windows users: use PowerShell, not the old Command Prompt. Most commands assume PowerShell.

1. Get the code

Download samplecode.zip from Piazza. When you unzip it, the folder inside is named versus_skeleton, that's the one you'll work in.

macOS

Safari unzips automatically. Move the resulting versus_skeleton folder to your Desktop so the path is short and predictable.

Windows

Right-click the downloaded zip in File Explorer → Extract All… → choose your Desktop. Important: Windows often creates an extra folder layer. After extracting you may have Desktop\versus_skeleton\versus_skeleton\app.py. If so, drag the inner versus_skeleton up one level so app.py sits directly inside Desktop\versus_skeleton.

Now open a terminal and cd into the folder.

macOS

cd ~/Desktop/versus_skeleton

Windows (PowerShell)

cd $HOME\Desktop\versus_skeleton

Don't see app.py when you ls / dir? See nested folder fix.

2. Install packages

The skeleton needs three Python libraries listed in requirements.txt: Flask, the MySQL connector, and Flask-Login.

2a. Activate your virtual environment

Reuse the cs460env from Lab 2. If your terminal prompt already starts with (cs460env), skip to 2b.

macOS

source ~/Desktop/cs460env/bin/activate

If you put the venv somewhere else in Lab 2, use that path. If you can't find it, just make a new one:

python3 -m venv ~/Desktop/cs460env
source ~/Desktop/cs460env/bin/activate

Windows (PowerShell)

$HOME\Desktop\cs460env\Scripts\Activate.ps1

If you don't have one yet:

python -m venv $HOME\Desktop\cs460env
$HOME\Desktop\cs460env\Scripts\Activate.ps1

Your prompt should now start with (cs460env).

PowerShell error "running scripts is disabled"? See execution policy fix.

2b. Install the packages

pip install -r requirements.txt

Got externally-managed-environment? See venv not active. Build/wheel error? See build error fix. "pip not found"? See pip fix.

3. Create the database

The skeleton ships with schema.sql, which creates a database called versus with four tables. Pick one of the two methods below.

Heads up

The first line of schema.sql is DROP DATABASE IF EXISTS versus;. If you already have a database called versus, running this will delete it and everything in it. That's fine for a fresh start, but don't run this against a database you care about.

Method A. Workbench (works the same on Mac and Windows)

  1. Open Workbench and connect to your local server (the Local instance tile, same as Lab 2).
  2. Top menu: File → Open SQL Script… and pick schema.sql from the versus_skeleton folder.
  3. Click the lightning-bolt button at the top of the editor to run the whole script.
  4. Right-click SCHEMAS in the Navigator → Refresh All. versus appears.

Method B. Terminal

macOS

mysql -u root -p < schema.sql

Type your MySQL root password (or press Enter if it's blank).

Windows (PowerShell)

PowerShell does not support < for input redirection. Use this instead:

Get-Content schema.sql | mysql -u root -p

3c. Confirm the tables exist

Open Workbench OR connect from the terminal:

mysql -u root -p

Then run:

USE versus;
SHOW TABLES;

Four tables should be listed: Brackets, Entrants, Matchups, Users.

"mysql is not recognized" / "command not found"? See Lab 2 PATH fix. PowerShell complains about <? See PowerShell redirection fix. "Access denied" loading the schema? See wrong password fix. SHOW TABLES empty? See empty database fix.

4. Run the app

4a. Edit your MySQL password into app.py

Open app.py in any text editor (VS Code, Sublime, even TextEdit if you set Format → Make Plain Text). Near the top, you'll see:

DB_USER     = 'root'
DB_PASSWORD = 'CHANGE_ME'
DB_NAME     = 'versus'
DB_HOST     = 'localhost'

Replace 'CHANGE_ME' with your actual MySQL root password. Keep the surrounding single quotes.

  • If your password is cs460pw: DB_PASSWORD = 'cs460pw'
  • If your password is blank (Lab 2 Homebrew default): DB_PASSWORD = '' (two single quotes, nothing between)
  • If your password contains a single quote like it's: use double quotes instead → DB_PASSWORD = "it's"

Save the file.

4b. Run the app

Back in your terminal (still inside the venv, still in the versus_skeleton folder):

python app.py

On macOS if python isn't found, use python3 app.py. (Inside an activated venv, both work.)

The output ends with Running on http://127.0.0.1:5000 and the terminal stays running. Leave it alone, the server is up.

Errors? ModuleNotFoundError, database access denied, unknown database 'versus', port 5000 in use, macOS AirPlay using port 5000, TemplateNotFound.

5. Try it in the browser

Open a browser and go to http://localhost:5000. The VERSUS home page should load.

Click through these in order:

  1. Click Register. Pick a username, use any email (the browser checks the @ format), pick a password. Submit.
  2. You should land on a "welcome to VERSUS" page logged in as your new user.
  3. Click Create a bracket. Give it a title. Leave entrant count at 8. Fill in all 8 entrant names (any words work). Submit.
  4. You should land on the bracket page showing 7 matchups total: 4 in round 1 (with both entrants filled in), 2 in round 2 (TBD), 1 in round 3 (TBD).
  5. Click Browse brackets. Your bracket appears in the list.

If register, create-bracket, and browse all worked, Lab 3 is complete.

To stop the server: switch to the terminal running python app.py and press Ctrl+C.

Page won't load? browser shows nothing. Register fails? apostrophe in username. Create bracket fails? missing entrant fields.

6. How forms work

You'll be writing more forms as you add features to VERSUS (predictions, votes, comments). The pattern is small and worth understanding now.

The HTML side

Open templates/register.html. It's a form with three input fields:

<form action='/register' method='post'>
  <input type='text'     name='username' required />
  <input type='email'    name='email'    required />
  <input type='password' name='password' required />
  <input type='submit'   value='Register' />
</form>

action='/register' says where to send the form data. method='post' says how. The name on each input is the key Flask uses to read that field.

The Flask side

In app.py, the register_user() function handles the POST:

username = request.form.get('username')
email    = request.form.get('email')
password = request.form.get('password')

Each request.form.get('xxx') matches an name='xxx' in the HTML. That's the whole link between the two files.

Same pattern everywhere: HTML form with names, Flask route reads them with request.form.get.

More: HTML forms (w3schools), Flask quickstart.

You're done when…

  1. SHOW TABLES; on the versus database lists Brackets, Entrants, Matchups, Users.
  2. python app.py prints "Running on http://127.0.0.1:5000" with no errors.
  3. You registered an account and landed on the welcome page.
  4. You created an 8-entrant bracket and the view page showed 7 matchups.

All four green? You're set. PA1 builds on this skeleton, your README lists what comes next.

Troubleshooting

Find your error below. Anything MySQL-only is in Lab 2 Troubleshooting.

Step 1: I cd'd in but don't see app.py

Windows often creates a double-nested folder when extracting (versus_skeleton\versus_skeleton\app.py). Run dir, and if you see another versus_skeleton folder, cd into it: cd versus_skeleton. Now app.py should be there.

Step 2: PowerShell says "running scripts is disabled on this system"

PowerShell blocks venv activation by default. Allow it for your user only (safe):

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Type Y and press Enter when it asks. Then try the activate command again. You only need to do this once per machine.

Step 2: "pip not found" or "pip is not recognized"

Your venv isn't activated. Re-run the activate command, confirm your prompt shows (cs460env), then retry. If pip is still missing inside the venv, use python -m pip install -r requirements.txt.

Step 2: "error: externally-managed-environment"

You ran pip outside the venv. Activate it (see 2a) so your prompt shows (cs460env), then retry the install.

Step 2: pip says "building wheel failed" or compilation errors

Brand-new Python releases sometimes lack pre-built wheels for the MySQL connector. Force binary-only install:

pip install --only-binary :all: -r requirements.txt

If that still fails, install Python 3.12 from python.org and rebuild the venv with python3.12 -m venv ... instead.

Step 3: PowerShell error "The '<' operator is reserved"

PowerShell doesn't support < for input redirection. Use the pipe form:

Get-Content schema.sql | mysql -u root -p

Or just load the file in Workbench (Method A in Step 3).

Step 3: "Access denied for user 'root'" when loading the schema

Wrong MySQL root password. Retype carefully. If you genuinely forgot it, reinstall MySQL with a memorable password (Lab 2 Troubleshooting under "Access denied").

Step 3: SHOW TABLES is empty

You're connected but not inside versus. Run USE versus; first, then SHOW TABLES;. If SHOW DATABASES; doesn't list versus at all, the schema didn't actually run, scroll back through the load output for the real error.

Step 4: "ModuleNotFoundError: No module named 'flask'" or "mysql"

The venv isn't active, or you installed into the wrong Python. Re-activate it, reinstall, then run:

source ~/Desktop/cs460env/bin/activate            # macOS
$HOME\Desktop\cs460env\Scripts\Activate.ps1       # Windows PowerShell
pip install -r requirements.txt
python app.py

Step 4: "mysql.connector.errors.ProgrammingError: 1045 Access denied"

The password in DB_PASSWORD doesn't match your real MySQL root password. Open app.py, fix the password between the single quotes, save, re-run. A blank password is DB_PASSWORD = '' (two quotes, nothing between).

Step 4: "Unknown database 'versus'"

The schema didn't run. Go back to Step 3. Confirm with SHOW DATABASES; that versus exists before retrying.

Step 4: "Address already in use" on port 5000

Either you already have a copy of the app running in another terminal (find it, Ctrl+C), or change the port. Open app.py, find the last line, change app.run(port=5000, debug=True) to app.run(port=5001, debug=True), save, re-run, and use http://localhost:5001 in the browser.

Step 4 (macOS): port 5000 used by AirPlay Receiver

macOS Monterey+ uses port 5000 for AirPlay Receiver. Two options:

  • Disable AirPlay Receiver: System Settings → General → AirDrop & Handoff, turn off AirPlay Receiver.
  • Or change Flask to a different port, same as the entry above.

Step 4: "jinja2.exceptions.TemplateNotFound: hello.html"

You ran python app.py from the wrong folder. The templates/ folder must sit right next to app.py in your current directory. cd back into the versus_skeleton folder and try again.

Step 5: Browser shows "This site can't be reached" or blank

Three things to check, in order:

  1. Is python app.py still running in your terminal? If it returned to the prompt, the server crashed, scroll up for the error.
  2. You're using http://localhost:5000, not https://. (No s.)
  3. If you changed the port to 5001 in app.py, use http://localhost:5001.

Step 5: Register fails with "You have an error in your SQL syntax"

You used an apostrophe in your username or password (e.g. O'Brien). The skeleton uses simple string-interpolated SQL which breaks on quotes, this is a known limitation of the teaching code, we'll fix it later in the course. For now use a username without single quotes.

Step 5: Create-bracket form refuses to submit

You changed the entrant count dropdown but didn't fill in every numbered seed field. The browser is silently blocking submission because some required fields are empty. Scroll the form and make sure every Seed line has text.

How do I stop the server?

Switch to the terminal running python app.py and press Ctrl+C. If that doesn't work (rare on Windows): close the whole terminal window.

I edited app.py but my changes aren't showing

Debug mode auto-reloads on most changes, but big edits (changing imports, adding routes) sometimes need a manual restart. Ctrl+C the server, run python app.py again, reload the browser.
